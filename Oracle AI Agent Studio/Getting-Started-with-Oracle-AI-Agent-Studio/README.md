## 🔍 Overview

Oracle Fusion AI Agent Studio enables you to build **intelligent, workflow-driven AI agents** that integrate seamlessly with enterprise processes across:

- Finance (FIN)
- Human Capital Management (HCM)
- Supply Chain Management (SCM)

With this platform, you can build AI Agents to:
- Automate business processes
- Build conversational workflows
- Integrate with REST APIs
- Create scalable **agentic applications**

---

## ✅ Prerequisites

Before starting, you should have:

- Basic understanding of Oracle Fusion business processes
- Knowledge of **security roles and data access**
- Familiarity with **JSON and REST APIs**
- Basic awareness of **AI / LLM concepts**

---

## 🧠 Core Concepts

### 🔐 Security & Roles
- Understand role-based access in Oracle Fusion
- Ensure agents operate within defined permissions

### ✍️ Prompt Engineering
A strong prompt should include:
- Clear instruction
- Context
- Constraints
- Expected output format

### 🤖 LLM Basics
- What are Large Language Models (LLMs)
- How they generate responses
- Limitations (hallucinations, context dependency)

### 🛡️ Guardrails
- Define:
  - Allowed actions
  - Restricted data access
  - Output validation rules

## 🧭 Learning Paths

### 👨‍💼 Functional Consultant Path

Focus on translating business requirements into AI-driven workflows and below learning path can be followed.

#### 🌐 REST API Fundamentals
- HTTP Methods: `GET`, `POST`, `PATCH`, `DELETE`
- Query parameters
- Authentication basics

#### 🧾 JSON & Dynamic Payloads
- Understand API request/response structure
- Build dynamic inputs and mappings

#### 🧩 AI Agent Studio Basics
- Business Objects
- Tools (REST, Document, etc.)
- Supervisor Agent Teams

#### 🔄 Workflow Agents
- Build workflows using nodes:
  - AI Node
  - Decision Node
  - Action Node

#### 🏗️ End-to-End Use Cases
- Invoice processing
- Employee query automation
- Approval workflows

#### 📊 Evaluation & Metrics
- Measure:
  - Accuracy
  - Performance
  - Response quality

#### ⚠️ Error Handling
- Retry mechanisms
- Fallback logic
- Success and failure responses

#### 🤖 Agentic Applications
- Multi-agent collaboration
- Enterprise-scale automation

---

### 👨‍💻 Technical Consultant Path

Focus on architecture, extensibility, and observability.

#### 🔗 MCP & A2A Protocols
- Model Context Protocol (MCP)
- Agent-to-Agent communication

#### 📡 Agent Observability
- Monitor:
  - Execution flows
  - API calls
  - Failures

#### 💻 JavaScript (Optional)
- Custom logic inside agents
- Data transformation
- Advanced workflow control

---

## 🪜 Step-by-Step Learning Roadmap

Follow this structured approach:

1. Learn **LLMs + Prompt Engineering + Guardrails**
2. Understand **Tools and Business Objects**
3. Build **basic AI agents**
4. Move to **Workflow Agents**
5. Develop **end-to-end use cases**
6. Implement **evaluation and monitoring**
7. Explore **advanced architectures (Agentic Apps, MCP, A2A)**
